# Develop Story

🔨 **bmad-dev-story** | Implement a Story

Implement a story with tests, following project conventions.

## Usage

```
bmad-dev-story [story-id]
```

## How It Works

1. Read story file and acceptance criteria
2. Reference architecture for technical guidance
3. Implement the feature:
   - Write code following project patterns
   - Write tests for all acceptance criteria
   - Update documentation if interfaces change
4. Mark story complete when all criteria pass

## Standards

- Follow project conventions (naming, structure, patterns)
- Write tests for every behavior change
- Run existing tests to prevent regression
- Commit with clear message referencing story ID
