## 🏗️ Winston (Architect)

Concordo que precisamos decidir agora. Mudar arquitetura de telemetria pós-launch é 10× mais caro. Duas abordagens:

### Opção A: Client-side beacon (lightweight)

```ts
// src/lib/telemetry.ts
export function track(event: string, payload: Record<string, unknown>) {
  if (process.env.NODE_ENV === "production") {
    navigator.sendBeacon("/api/telemetry", JSON.stringify({ event, payload, ts: Date.now() }));
  }
}
```

- **Prós:** Simples, não bloqueia UI, funciona offline (buffered)
- **Contras:** Usuário pode bloquear, dados são perdidos se beacon falhar, não dá retry

### Opção B: Queue + Batch (robusto)

```ts
// src/lib/telemetry/queue.ts
class TelemetryQueue {
  private buffer: TelemetryEvent[] = [];
  private flushInterval = 5000; // 5s

  push(event: TelemetryEvent) { this.buffer.push(event); }
  async flush() { /* POST batch to API */ }
}
```

- **Prós:** Retry, batching, resiliência
- **Contras:** Mais complexo, precisa de serviço backend

### Opção C: Third-party analytics (PostHog / Amplitude / Mixpanel)

- **Prós:** Pronto, dashboards, funnels, cohorts
- **Contras:** Custo ($$$), vendor lock-in, dados saem da nossa infra

Minha recomendação: **Opção B** para dados de trade (sensíveis, queremos controle) + **Opção C** para product analytics (funnels, onboarding drop-off).

## 🏗️ Winston (Architect)

Sobre o `maxDrawdownPercent` — isso exige mudança no `tick` loop. Hoje o `checkPosition` só verifica se fechou. Precisaríamos de um `PositionTracker` que, a cada tick, calcula:

```ts
const unrealizedPnlPercent = ((currentPrice - entry) / entry) * leverage * 100;
const drawdown = Math.min(0, unrealizedPnlPercent); // só negativo
maxDrawdown = Math.min(maxDrawdown, drawdown);
```

Isso é leve — 2 operações por tick — mas precisa ser feito no engine, não no slice.

**Outro ponto arquitetural:** Os `console.log` que restauramos são úteis para debug, mas **devem ser stripped em produção**. Hoje não fazemos tree-shaking de logs. Sugiro:

```ts
// src/lib/logger.ts
const isDev = process.env.NODE_ENV !== "production";
export const logger = {
  log: (...args: unknown[]) => { if (isDev) console.log(...args); },
  error: (...args: unknown[]) => { if (isDev) console.error(...args); },
};
```

Substituir todos os `console.log` no store/engine por `logger.log()`. Em produção, zero logs no console. Telemetria vai para o backend.
